# 📜 GHS Protocol Rules

- **Bilingual Documentation**: `HISTORY.md` and `BUGS.md` must be maintained in English and Spanish as per configuration.
- **Table Integrity**: Ensure all tables in `HISTORY.md` and `BUGS.md` preserve their structure (Commit, Author, Description, Details).
- **Trigger Tags**:
  - Use `#ai-history` for technical summaries.
  - Use `#ai-bug` for error registry.
  - Use `#ai-catchup` for batch processing.
- **Semantic Details**: Always include specific technical details (filenames, functions modified) in the "Technical Details" column.
